Memory throughput Utilization

Shared memory and caches (i.e., L1/L2 caches available on devices compute capability 2.x and higher) should be used to minimized memory transfers and global memory accesses. The typical patter of a program that use shared memory is :

  • Load data in shared memory. Usually each thread in a block load its address.

  • Synchronize with the other threads so the memory is coherent after its population.

  • Process data in shared memory as much as possible.

  • Write back to device memory.

As explained in section [memoryModel] accesses pattern is important even in shared memory, so a second phase of optimization is often necessary to organize accesses in order to enable coalesced access in case of global memory 43 or avoid bank conflict in shared memory.
