Constant Memory

Specific part of device memory is the constant memory, which allows to store limited amount (64KB) of constant symbols. Similarly to the texture memory, the accesses are cached but only reading is allowed. Constant memory should be used for small variables that are shared among all threads and do no require interpolation. It is declared by the host using the keyword __constant__, and its size is to be known at compile time. There are two main reason why reading from constant memory can save bandwidth:

  • A single read can be broadcasted to all thread of the same warp .

  • Constant memory is cached, so consecutive reads of the same address will not incur any additional traffic on off-chip DRAM.
